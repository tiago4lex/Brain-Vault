# Oque é um Endereço IP?

Um **endereço IP *(Internet Protocol)*** é um identificador único atribuído a cada dispositivo em uma rede, permitindo o **envio e recebimento de dados entre eles**.

Existem dois principais tipos:
- **IPv4 *(Internet Protocol Version 4)***
- **IPv6 *(Internet Protocol Version 6)***

---
# Ipv4 - *Internet Protocol Version 4*

## Características

- **Comprimento:** 32 bits
- **Formata:** decimal com 4 blocos (ex: `192.168.1.1`)
- **Número total de endereços possíveis:** aproximadamente 4,3 bilhões
- **[[2.3 Classes de Endereço IP]]:** A, B, C, D, E (antigo modelo de classificação)

## Exemplo

```makefile
IP:      192.168.1.1
Binário: 11000000.10101000.00000001.00000001
```

## Limitações

- Quantidade limitada de endereços (esgotamento)
- Necessidade de NAT *(Network Address Translation)* para múltiplos dispositivos
- Segurança limitada por padrão

---
# IPv6 - *Internet Protocol Version 6*

## Características

- **Comprimento:** 128 bits
- **Formata:** hexadecimal com 8 grupos de 4 dígitos (ex: `2001:0db8:85a3:0000:0000:8a2e:00370:7334`)
- **Número total de endereços possíveis:** 340 undecilhões (340x10<sup>36</sup>)
- **Desenvolvido para substituit o IPv4**

## Exemplo

```yaml
Endereço completo: 2001:0db8:0000:0000:0000:0000:1428:57ab
Forma reduzida:    2001:db8::1428:57ab
```

## Vantagens do IPv6

- Quantidade praticamente **infinita** de endereços
- **Sem necessidade de NAT**
- **Segurança embutida** (IPSec nativo)
- Suporte a **autoconfiguração** e mobilidade
- Melhoria na **eficiência de roteamento**

---
# Comparação IPv4 vs IPv6

| **Características**      | **IPv4**                    | **IPv6**                          |
| ------------------------ | --------------------------- | --------------------------------- |
| Tamanho do endereço      | 32 bits                     | 128 bits                          |
| Formato                  | Decimal (ex: `192.168.1.1`) | Hexadecimal (ex: `2001:db8::1`)   |
| Nº de endereços          | ~4,3 bilhões                | 340 undecilhões                   |
| Segurança embutida       | Não                         | Sim (IPSec)                       |
| NAT necessário?          | Sim                         | Não (em muitos casos)             |
| Autoconfiguração         | Limitada                    | Sim (stateless & DHCPv6)          |
| Eficiência de roteamento | Menor                       | Maior                             |
| Implementação atual      | Ampla                       | Crescente, mas ainda em transição |

---
# Como identificar e configurar IPv4 e IPv6

## Windows

```bash
ipconfig /all
```

## Linux

```bash
ip a
```

## Configuração manual (exemplo IPv6 no Linux)

```bash
sudo ip -6 addr add 2001:db8::1/64 dev eth0
```

# Tipos de endereços IPv6

| **Tipo**       | **Prefixo** | **Uso**                         |
| -------------- | ----------- | ------------------------------- |
| Unicast        | -           | Endereço único por dispositivo  |
| Multicast      | FF00::/8    | Envia para um grupo de nós      |
| Anycast        | -           | Envia para o mais próximo       |
| Link-local     | FE80::/10   | Comunicação local sem roteador  |
| Global unicast | 2000::/3    | Internet pública                |
| Loopback       | ::1         | Dispositivo se comunica consigo |

---
# Por que o IPv6 é necessário?

- O número de dispositivos conectados à internet cresce rapidamente (IoT, celulares, PCs, etc.)
- O IPv4 <font color="#ffff00"><strong>não tem endereços suficientes</strong></font> para todos.
- O IPv6 foi criado para ser escalável, eficiente e seguro

---
# IPv6 no mundo real

- Todos os sistemas operacionais modernos já suportam IPv6
- Muitos roteadores domésticos e ISPs já oferecem suporte
- Transição ainda em andamento: <font color="#ffff00"><strong>IPv4 e IPv6 coexistem</strong></font> (pilha dupla)

---
# Conlusão

- O IPv4 ainda é amplamente utiliziado, mas está limitado pela quantidade de endereços
- O IPv6 representa o futuro da internet: mais escalável, seguro e eficiente
- Profissionais de redes devem conhecer ambos os protocolos para atual no mercado atual futuro.
