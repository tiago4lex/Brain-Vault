# O que é uma Máscara de Rede?

A **máscara de rede** (ou *subnet mask*) é um conjunto de números que <font color="#f79646"><strong>define a parte do endereço IP que corresponde à rede e a parte que corresponde aos hosts</strong></font> (dispositivos conectados).
Ela é essencial para que dispositivos em uma rede consigam se comunicar corretamente e para dividir redes grandes em **sub-redes menores** (subnetting).

----
# Exemplo de Endereço IP e Máscara

- Endereço IP: `198.168.1.10`
- Máscara de rede: `255.255.255.0`

Em binário
```makefile
IP:      1000000.10101000.000000001.00001010
Máscara: 111111111.111111111.111111111.000000000
```

## Interpretação:

- Os **bits 1** da máscara indicam a <font color="#8db3e2"><strong>parte da rede</strong></font> (`198.168.1`)
- Os **bits 0** indicam a <font color="#c3d69b"><strong>parte do host</strong></font> (dispositivos finais: `0 a 255`)

Ou seja: todos os IPs de `192.168.1.1` até `192.168.1.254` estão na **mesma rede local**.

---
# Como Funciona

A máscara de rede <font color="#d99694">separa o IP em dois blocos</font>:
- **Identificador de Rede** (network ID)
- **Identificador de Host** (host ID)

O roteador usa essa informação para saber:
- Se um pacote de dados vai para um **dispositivo na mesma rede**
- Ou se deve ser enviado para **outra rede via gateway**

# CIDR: Forma Abreviada da Máscara

Em vez de escrever `255.255.255.0`, podemos usar **CIDR**:
- `192.168.0.0/24` -> indica que os **24 primeiros bits** são da rede

Outros exemplos:
- `/16` -> 255.255.0.0
- `/30` -> 255.255.255.252

# Como configurar a Máscara de Rede

## Windows

1. Painel de Controle > Rede e Internet > Conexões de Rede
2. Clique com o botão direto > Propriedades
3. Selecione "Protocolo TCP/IPv4"
4. Configurar IP, Máscara e Gateway manualmente

**Pelo prompt de comando:**

```bash
netsh interface ip set address "Ethernet" static 192.168.1.10 255.255.255.0 198.168.1.1
```

## Linux

```bash
sudo ifconfig eth0 192.168.1.10 netmask 255.255.255.0
```

ou

```bash
sudo ip addr add 192.168.1.10/24 dev eth0
```

## Roteadores

Via interface web (geralmente 192.168.0.1):
- Atribuir faixa de IPs
- Definir máscara de sub-rede
- Dividir redes com DHCP

# Para que serve *subnetting*?

O processo de ***subnetting*** (subdivisão de redes) é útil para:
- Organizar redes por setores (ex: RH, TI, Financeiro)
- Melhorar a segurança e o desempenho
- Reduzir o tráfego de broadcast
- Economizar endereços IP

---
# Resumo

- A máscara de rede define <font color="#ffc000"><strong>onde termina o endereço e onde começa o endereço do host</strong></font>.
- Trabalha junto com o IP para garantir a comunicação correta.
- *Subnetting* permite criar várias redes lógicas dentro de uma rede maior.
