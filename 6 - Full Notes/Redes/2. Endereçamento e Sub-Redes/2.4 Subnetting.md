# O que é *Subnetting*?

***Subnetting*** (subdivisão de redes) é o processo de <font color="#4bacc6"><strong>dividir uma rede IP maior em múltiplas sub-redes menores</strong></font>.Isso melhora:

- O uso eficiente de endereços IP.
- Organização de rede.
- A segurança e controle de tráfego.

---
# Conceitos Importantes

- **Endereço IP:** 32 bits, dividido em 4 octetos
- **Máscara de sub-rede:** define quantos bits são usados para a **rede** e quantos para **hosts**
- **CIDR *(Classless Inter-Domain Routing)*:** notação que define a quantidade de bits da máscara. Ex `/24`
---
# Fórmulas para Subnetting

1. **Números de sub-rede:** 2<sup>n</sup> , onde **n** = bits "emprestados" dos hosts.
2. **Número de hosts por sub-rede:** 2<sup>h</sup> , onde **h** = bits restantes para host. (subtrai 2 para rede e broadcast).
3. **Incremento:** 256 - último octeto da máscara.

---
# Exemplo 1 - Dividir uma rede Classe C

## Objetivo

Dividir a rede `192.168.1.0/24` em **4 sub-redes**.

## Passo a Passo:

1. **Máscara original:** /24 = 255.255.255.0 → 8 bits disponíveis para hosts (último octeto)

2. Para 4 sub-redes → precisamos de 2 bits:
	- 2<sup>2</sup> = 4 sub-redes

3. Nova máscara: /24 + 2 = /26
	- 255.255.255.192
	- 6 bits para host

4. **Hosts por sub-rede:** 2<sup>6</sup> - 2 = **62**
5. **Incremento:** 256 - 192 = 64

## Resultado

| **Sub-rede** | **Faixa de IPs** | **Broadcast** | **Hosts válidos**             |
| ------------ | ---------------- | ------------- | ----------------------------- |
| 1            | 192.168.1.0/26   | 192.168.1.63  | 192.168.1.1 - 192.168.1.62    |
| 2            | 192.168.1.64/26  | 192.168.1.127 | 192.168.1.65 - 192.168.1.126  |
| 3            | 192.168.1.128/26 | 192.168.1.191 | 192.168.1.129 - 192.168.1.190 |
| 4            | 192.168.1.192/26 | 192.168.1.255 | 192.168.1.193 - 192.168.1.254 |

# Exemplo 2 – Classe B com /16

## Objetivo:

Dividir a rede `172.16.0.0/16` em **100 sub-redes**.

## Passo a Passo:

1. Classe B padrão = /16 → 16 bits disponíveis

2. Quantos bits para criar 100 sub-redes?  
    2⁷ = 128 → usamos 7 bits

3. Nova máscara = /16 + 7 = **/23**  
    → 255.255.254.0  
    → 9 bits restantes para host

4. Hosts por sub-rede: 2⁹ - 2 = **510**

5. Incremento: 256 - 254 = **2** no 3º octeto

# Exemplo 3 - Converter máscara para CIDR

## Objetivo:

Descobrir o número de bits a partir da máscara de sub-rede `255.255.240.0`

## Passo a Passo

1. Converta cada octeto para binário:

| **Octeto** | Binário  |
| ---------- | -------- |
| 255        | 11111111 |
| 255        | 11111111 |
| 240        | 11110000 |
| 0          | 00000000 |

2. Conte os bits "1"

```
11111111.11111111.11110000.00000000
   8    +   8    +   4    +   0      =  **20 bits**
```

**Resultado:** CIDR = /20

## Exemplo de sub-redes:

| Sub-rede      | Broadcast    | Faixa de Hosts            |
| ------------- | ------------ | ------------------------- |
| 172.16.0.0/23 | 172.16.1.255 | 172.16.0.1 – 172.16.1.254 |
| 172.16.2.0/23 | 172.16.3.255 | 172.16.2.1 – 172.16.3.254 |
| ...           | ...          | ...                       |

# Dicas de Subnetting

- Use **incremento** para encontrar o próximo bloco.

- Use calculadoras de subnet para validar resultados.

- Lembre-se que o primeiro IP é **rede**, e o último é **broadcast**.

# Conclusão

O **subnetting** é essencial para planejar e organizar redes. Ele garante o uso eficiente dos endereços IP, melhora a segurança e desempenho da rede e permite uma melhor administração de tráfego.

Saber **calcular sub-redes manualmente** é importante para certificações como **CCNA** e também para entender como funciona o roteamento e design de redes.
