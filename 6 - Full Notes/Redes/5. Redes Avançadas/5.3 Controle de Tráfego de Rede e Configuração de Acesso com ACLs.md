# O que é Tráfego de Rede?

Tráfego de rede refere-se ao **fluxo de dados que circula entre dispositivos em uma rede**. Esse tráfego pode incluir:

- **Requisições e respostas HTTP/HTTPS** (navegação web)
- **Transmissões de vídeo ou áudio**
- **Transferência de arquivos**
- **Consulta DNS**
- **Dados de autenticação, controle e monitoramento**

Existem diferentes **tipos de tráfego**:

- **Unicast:** de um dispositivo para outro específico.
- **Broadcast:** enviado para todas os dispositivos da rede.
- **Multicast:** enviado para um grupo específico de dispositivos.

---
# Por que controlar o tráfego?

Controlar o tráfego é essencial para:

- **Segurança:** impedir acessos não autorizados.
- **Performance:** evitar congestionamento e priorizar serviços críticos.
- **Gerenciamento:** definir o que é permitido ou bloqueado.

---
# Como funciona  a configuração de permissões de acesso no Cisco?

No Cisco, as permissões de tráfego são controladas por **ACLs *(Access Control List)***, que são regras aplicadas a interfaces de rede que **permitirão ou negarão tráfego com base em critérios como IP, porta e protocolo**.

## Tipos de ACLs

- **ACL *Standard*:** Filtra tráfego apenas com base no endereço IP de origem.
- **ACL *Extended*:** Filtra por **IP de origem/destino, protocolos (TCP/UDP/ICMP), portas (HTTP/DNS/ etc.)**.

# Exemplo de Configuração no Cisco (via CLI)

## Objetivo

Permitir que apenas o **PC1** acesse o **servidor Web (192.168.2.2)** e bloquear todos os outros.

## 1. Criar a ACL

```text
Router> enable
Router# configure terminal

Router(config)# access-list extended [nome_da_lista]
Router(config-ext-nacl)# permit tcp 192.168.1.2 0.0.0.0 host 192.168.2.2.2 eq 80
Router(config-ext-nacl)# deny tcp any host 192.168.2.2 eq 80
Router(config)# permit ip any any
```

### **Explicação**

- Primeira linha: Permite que **192.168.1.2** (PC1) acesse **192.168.2.2** (Servidor Web) via **porta 80 (HTTP)**.
- Segunda linha: Bloqueia qualquer outro tráfego com destino à porta 80 do servidor.
- Terceira linha: permite o restante do tráfego da rede.

## 2. Aplicar a ACL na interface

Supondo que a interface **FastEthernet0/0** é a que conecta os PCs à rede:

```text
Router(config)# interface Fa 0/0
Router(config-if)# ip access-group [nome_da_lista] in
Router(config-if)# exit
Router(config)# exit
```

## 3. Verificando a ACL

```text
Router# show access-list
```

---
# Considerações importantes

- ACLs são lidas **de cima para baixo.** A primeira correspondência define o que acontece.
- Se não houver nenhuma regra aplicável, o tráfego é negado por padrão.
- ACLs aplicadas no modo `in` afetam o tráfego **que entra na interface**, e `out` o que sai.
