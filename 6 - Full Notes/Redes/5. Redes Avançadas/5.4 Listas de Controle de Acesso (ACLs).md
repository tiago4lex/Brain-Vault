# Introdução

## Oque são ACLs

**ACL *(Access Control List)*** é uma lista de regras que define quais pacotes de dados são permitidos ou negados ao passar por uma interface de rede. As ACLs funcionam como um **filtro de tráfego** com base critérios como:

- Endereço IP de origem ou destino
- Tipo de protocolo (TCP, UDP, ICMP, etc.)
- Porta (HTTP 80, DNS 53, etc.)

---
# Para que servem as ACLs?

As ACLs são usadas para:

- **Restringir o acesso** a dispositivos ou redes específicas
- **Reduzir tráfego desnecessário**
- **Aumentar a segurança**
- **Controlar rotas e acesso a serviços específicos**

---
# Tipos de ACLs


| **Tipo**               | **Descrição**                                                 |
| ---------------------- | ------------------------------------------------------------- |
| Padrão *(Standard)*    | Filtra tráfego apenas com base no **IP de origem**            |
| Estendida *(Extended)* | Filtra por **IP de origem e destino, protocolo e portas**     |
| Nomeadas *(Named)*     | ACLs com **nomes descritivos**, podem ser padrão ou estendida |

---
# Sintaxe Básica

## ACL Padrão

```bash
access-list [NÚMERO] [permit/deny] [IP origem] [wildcard]
```

## ACL Estendida

```bash
access-list [NÚMERO] [permit/deny] [protocolo] [IP origem] [wildcard] [IP destino] [wildcard] [porta]
```

> O ***wildcard mask*** é o "inverso" da máscara de sub-rede.
> Exemplo: Máscara `255.255.255.0` → Wildcard `0.0.0.255`

---
# Exemplo prático - ACL Estendida

## Objetivo

Permitir apenas o **PC1 (192.168.1.2)** a cessar o **Servidor Web (192.168.2.2)** na **porta 80 (HTTP)**, e bloquear todos os outros.

## 1. Criar ACL

```bash
enable
configure terminal

access-list 100 permit tcp host 192.168.1.2 host 192.168.2.2 eq 80
access-list 100 deny tcp any host 192.168.2.2 eq 80
access-list 100 permit ip any any
```

## 2. Aplicar ACL na interface

```bash
interface FastEthernet0/0
ip access-group 100 in
exit
```

> A ACL deve ser aplicada na interface de entrada ou saída, dependendo da direção do tráfego.

## Verificado ACLs

- Para listar todas as ACLs configuradas:

```bash
show access-lists
```

- Para verificar ACL aplicada em uma interface:

```bash
show running-config
```

---
# Dicas Importantes

- ACLs são lidas **de cima para baixo**: **a primeira correspondência é usada**.
- Se nenhum critério for correspondido, o tráfego **é negador por padrão *(implicit deny)***
- Evite colocar ACLs nas interfaces erradas ou sem uma regra de permissão para outros tráfegos (isso pode bloquear tudo).
- Use sem um `permit ip any any` no final se quiser permitir o restante do tráfego.

---
# Exemplo de ACL Padrão

```bash
access-list 10 deny 192.168.1.5 0.0.0.0
access-list 10 permit any

interface FastEthernet0/0
ip access-group 10 in
```

> Este exemplo bloqueia o IP `192.168.1.5` e permite o restante do tráfego.

---
# Conclusão

As ACLs são ferramentas fundamentais para **segurança e controle de tráfego em redes**. Com elas, é possível **especificar com precisão** quem pode acessar o quê e como.
