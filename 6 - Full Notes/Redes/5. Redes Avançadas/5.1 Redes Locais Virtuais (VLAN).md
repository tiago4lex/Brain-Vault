# O que são VLANs?

**VLAN** significa ***Virtual Local Area Network***, ou **Rede Local Virtual**. É uma tecnologia que permite segmentar uma rede física em várias redes **lógicas e isoladas**, mesmo que os dispositivos estejam conectados ao mesmo switch físico.

----
# Objetivos das VLANs

- **Separar tráfego de diferentes setores da empresa**
- **Melhorar segurança e desempenho**
- **Reduzir domínios de broadcast**
- **Facilitar a administração da rede**

---
# Como Funcionam?

Cada VLAN é como se fosse uma **sub-rede separada**. Computadores na VLAN 10, por exemplo, <font color="#4f81bd"><strong>não conseguem se comunicar com a VLAN 20 sem a ajuda de um roteador ou switch de camada 3</strong></font>.

Mesmo estando no mesmo switch, portas diferentes podem estar em VLANs diferentes.

## Exemplo

| **Porta do Switch** | **Dispositivo**       | **VLAN** |
| ------------------- | --------------------- | -------- |
| Fa0/1               | Computador do RH      | VLAN 10  |
| Fa0/2               | Computador da TI      | VLAN 20  |
| Fa0/3               | Computador Financeiro | VLAN 30  |

## Tipos de Porta

- **Access:** Porta atribuída a uma única VLAN. Ex: PC Comum.
- **Trunk:** Porta que transporta múltiplas VLANS (usada entre switches ou para roteadores).

---
# Como criar VLANS no **Cisco (Packet Tracer ou equipamento real)**

## 1. Entrar no modo de configuração

```bash
enable
configure terminal
```

## 2. Criar VLANs

```bash
vlan 10
name RH
exit

vlan 20
name TI
exit
```

## 3. Atribuir portas às VLANs

```bash
interface FastEthernet0/1
switchport mode acess
switchport acess vlan 10
exit

interface FastEthernet0/2
switchport mode acess
switchport acess vlan 20
exit
```

## 4. Verificar as configurações

```bash
show vlan brief
```

# Comunicação entre VLANs

Por padrão, <font color="#ff0000"><strong>VLANs não se comunicam entre si</strong></font>. Para permitir isso, usa-se:

- **Router-on-a-stick:** Um roteador com subinterfaces
- **Switch de camada 3:** Que faz roteamento entre VLANs

## Exemplo de router-on-a-stick

```bash
interface FastEthernet0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
exit
ip dhcp pool vlan10
default-router 192.168.10.1
exit

interface FastEthernet0/0.20
encapsulation dot1Q 20
ip address 192.168.20.1 255.255.255.0
exit
ip dhcp pool vlan20
default-router 192.168.20.1
exit
```

---
# Vantagens das VLANs

- **Segurança:** Usuários de setores diferentes não acessam a rede do outro diretamente.
- **Organização:** Facilita identificar e administrar setores.
- **Eficiência:** Reduz broadcast e melhora performance

---
# Aplicações Práticas

- Separar setores (RH, TI, Financeiro)
- Isolar dispositivos (como câmeras IP)
- Gerenciar convidados (VLAN Guest)

---
# Conclusão

As VLANs são fundamentais para redes escaláveis e seguras. Mesmo com um único switch, é possível ter várias redes isoladas logicamente, melhorando organização, segurança e desempenho.
