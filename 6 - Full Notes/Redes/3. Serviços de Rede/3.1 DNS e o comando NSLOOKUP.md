# O que é DNS?

## Definição

<font color="#de7802"><strong>DNS (Domain Name System)</strong></font> é um sistema que traduz nomes de domínios legíveis por humanos (como `www.google.com`) em endereços IP numéricos (como `142.250.218.100`),  que são utilizados pelas máquinas para se comunicarem na rede.

## Por que é necessário?

 - Os humanos preferem nomes fáceis de lembrar (ex: `youtube.com`), mas os computadores precisam de endereços IP para localizar servidores.
 - O DNS atua como uma "agenda telefônica da internet", associando nomas a IPs automaticamente.

## Como funciona?

1. O usuário digita um endereço no navegador.
2. O navegador pergunta ao sistema operacional qual é o IP correspondente.
3. O sistema consulta servidores DNS (geralmente configurados pelo provedor de internet).
4. Se o nome ainda não estiver em cache, o servidor DNS faz uma busca hierárquica até encontrar o IP.
5. O endereço IP é retornado ao navegador, que acessa o servidor correto.

## Componentes principais

- **Domínio:** nome completo como `openai.com`.
- **Servidor DNS:** responsável por armazenar registros de nomes e IPs.
- **Registros DNS:**
	- `A`: mapeia um domínio a um IPv4
	- `AAAA`: mapeia um domínio a um IPv6
	- `MX`: indica o servidor de e-mail
	- `CNAME`: define apelidos para domínios
	- `NS`: identifica os servidores responsáveis por aquele domínio


# Comando `NSLOOKUP`

## O que é?

O `NSLOOKUP` é uma ferramenta de linha de comando que permite consultar informações do DNS sobre domínios e endereços IP.

## Para que serve?

- Verificar o IP associado a um domínio
- Consultar registros DNS específicos (A, MX, CNAME, etc)
- Testar servidores DNS específicos
- Diagnosticar problemas relacionados à resolução de nomes

**Sintaxe básica**

```bash
nslookup [domínio]
```

**Exemplo:**

```bash
nslookup google.com
```

**Resultado:**

```bash
Servidor:    dns.google
Address:     8.8.8.8

Nome:        google.com
Addresses:   142.250.218.206
```

**Consultas avançadas:**

```bash
nslookup
> set type=mx
> gmail.com
```

### Parâmetros úteis:

- `set type=A` – busca por registros IPv4.
    
- `set type=AAAA` – busca por IPv6.
    
- `set type=MX` – busca por servidores de e-mail.
    
- `set type=NS` – busca por servidores de nomes.
    
- `server [ip]` – consulta usando um servidor DNS específico.

# Conclusão

Entender o funcionamento do DNS é essencial para quem estuda redes, pois ele é a base de quase toda navegação na internet. O comando `nslookup` é uma ferramenta poderosa para inspecionar e diagnosticar problemas relacionados a domínios, endereços IP e servidores de nomes.
