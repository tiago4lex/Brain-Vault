# O que é um Provedor de Serviços (ISP)?

Um **ISP *(Internet Service Provider)*** é a empresa ou infraestrutura que fornece **acesso à internet** para usuários finais (residências, empresas, etc.). Ele é responsável por:

- Entregar um **IP público** ao cliente.
- Encaminhar pacotes para destinos na Internet.
- Gerenciar o tráfego da rede global.
- Possuir rotas para as redes públicas.

---
# Como o ISP funciona na prática?

Quando um cliente quer acessar um site na Internet:

1. O dispositivo envia um pacote com IP de destino externo.
2. O roteador local (do cliente) aplica **NAT** (se necessário).
3. O pacote é encaminhado para o **roteador do ISP**.
4. O ISP, com suas **rotas e conexões de backbone**, envia esse pacote para a Internet.

---
# Simulando um ISP no Cisco Packet Tracer

No **Packet Tracer**, o ISP é **representado por um roteador**, às vees com um **servidor DNS/Web** atrás dele.

## Exemplo prático: configuração básico de ISP

**Objetivo:**

Simular uma rede onde um cliente (com IP privado) se conecta a um roteador ISP (com IP público) e consegue navegar na web.

## Topologia sugerida:

```java
PC → Switch → Roteador Cliente → Roteador ISP → Servidor Web (Internet)
```

## 1. Configuração do **roteador do cliente** (R1)

```bash
enable
configure terminal

interface g0/0
ip address 192.168.1.1 255.255.255.0
ip nat inside
no shutdown
exit

interface g0/1
ip address 200.1.1.2 255.255.255.252
ip nat outside
no shutdown
exit

access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 interface g0/1 overload

ip route 0.0.0.0 0.0.0.0 200.1.1.1
```

## 2. Configuração do **roteador do ISP** (R2)

```bash
enable
configure terminal

interface g0/0
ip address 200.1.1.1 255.255.255.252
no shutdown
exit

! Suponha que o "lado da Internet" esteja em g0/1
interface g0/1
ip address 8.8.8.8 255.255.255.0
no shutdown
exit

ip route 192.168.1.0 255.255.255.0 200.1.1.2
```

## 3. Configuração do **PC cliente**

No menu **Desktop > IP Configuration**:

- IP Address: `192.168.1.10`
- Subnet Mask: `255.255.255.0`
- Default Gateway: `192.168.1.1`
- DNS: `8.8.8.8` ou IP do servidor DNS na rede

## 4. Configuração do **servidor** (opcional)

- IP: `8.8.8.8`
- Ative o serviço Web (HTTP)
- Ative o serviço DNS (com nome como `www.site.com`)

---
# Como o ISP lida com os pacotes?

1. **Recebe** os pacotes NATeados do cliente.
2. **Encaminha** para o destino real (Internet, outro roteador, servidor, etc.).
3. Mantém **rotas estáticas ou dinâmicas** para alcançar redes externas.
4. Fornece o **IP público** para os clientes (manual ou via DHCP).

---
# Tipos de conexões que o ISP pode oferecer

| **Tipo de conexão** | **Descrição**                           |
| ------------------- | --------------------------------------- |
| IP Estático         | Cliente recebe IP público fixo          |
| IP Dinâmico         | Cliente recebe IP via DHCP/NAT          |
| PPPoE               | Conexão autenticada entre cliente e ISP |
| MPLS                | Usado por ISPs para redes privadas      |

---
# Recursos extras de um ISP no Packet Tracer

Além de um roteador, você pode simular com:

- **Servidor DNS:** resolve nomes de domínio.
- **Servidor Web:** hospedas sites acessíveis pelos clientes.
- **ACLs Firewalls:** para segurança do tráfego.
- **Roteadores com rotas BGP (avançado):** para simular múltiplos ISPs.

---
# Conclusão

Configurar um **provedor de serviços (ISP)** no Cisco Packet Tracer permite entender como 
funciona o acesso à Internet, o roteamento entre redes diferentes e o papel crucial do **NAT**. A comunicação entre cliente e ISP é essencial para permitir que máquinas com IPs privados naveguem na Internet por meio de um IP público fornecido pelo roteador do provedor.
