# O que é um `package`

Em java, um `package` (pacote) é uma forma de **organizar classes e arquivos** dentro de um projeto.

Ele funciona como **uma pasta** no sistema de arquivos, agrupando classes que têm **funções relacionadas**.

> Em resumo: `package` = pasta que organiza o código em grupos lógicos.


## Por que usar pacotes?

- **Organização:** facilita encontrar e gerenciar as classes do projeto.
- **Evita conflitos de nomes:** duas classes com o mesmo nome podem existir em pacotes diferentes.
- **Controle de acesso:** é possível definir a visibilidade de classes e métodos entre pacotes.
- **Manutenção facilitada:** ajuda a estrutura grandes sistemas de forma modular.

## Exemplo de pacote

Estrutura de pastas:

```css
src/
 └── br/
     └── com/
         └── user/
             └── projeto/
                 ├── Main.java
                 └── modelo/
                     └── Pessoa.java

```

**Dentro de `Pessoa.java`**

```java
package br.com.user.projeto.modelo;

public class Pessoa {
    public String nome;
    public int idade;

    public void exibirInformacoes() {
        System.out.println("Nome: " + nome + ", Idade: " + idade);
    }
}
```

**Dentro de `Main.java`:**

```java
package br.com.user.projeto;

import br.com.user.projeto.modelo.Pessoa; // importando a classe Pessoa

public class Main {
	public static void main(String[] args) {
		Pessoa p = new Pessoa();
		p.nome = "Maria";
		p.idade = 23;
		p.exibirInformacoes();
	}
}
```

# O que é o `import`

O **comando `import`** é usado para **trazer classes de outros pacotes** para o arquivo atual, permitindo seu uso direto.

Sem `ìmport`, seria necessário usar o **nome completo (caminho completo)** da classe.

## Exemplo com e sem `import`

- Sem usar `import`:

```java
package br.com.user.projeto;

public class Main {
	public static void main(String[] args) {
		br.com.user.projeto.modelo.Pessoa p = new br.com.user.projeto.modelo.Pessoa();
		p.nome = "Maria";
		p.idade = 23;
		p.exibirInformacoes();
	}
}
```

- Usando `import`:

```java
package br.com.user.projeto;

import br.com.user.projeto.modelo.pessoa; // Importa a classe

public class Main {
	public static void main(String[] args) {
		Pessoa p = new Pessoa(); // mais limpo e legível
		p.nome = "Maria";
		p.idade = 23;
		p.exibirInformacoes();
	}
}
```

# Tipos de Importação

## 1. Importação específica

Importa **apenas uma classe** de um pacote.

```java
import java.util.Scanner; // apenas Scanner
```

## 2. Importação com coringa (`*`)

Importa **todas as classes** de um pacote, **mas não importa subpacotes**.

```java
import java.util.*;  // importa todas as classes do pacote java.util
```

**Exemplo:**

```java
import java.util.*;

public class Exemplo {
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);
		ArrayList<String> lista = new ArrayList<>();
	}
}
```

>[!warning] **Atenção:**
>`import java.util.*` **não** importa classes de subpacotes, como `java.util.concurrent`.

## 3. Importação estática

Permite importar **métodos e atributos estáticos** diretamente, sem precisar escrever o nome da classe antes.

```java
import static java.lang.Math.*; // importa métodos e constantes da classe Math

public class Exemplo {
    public static void main(String[] args) {
        System.out.println(sqrt(16)); // usa sqrt diretamente
        System.out.println(PI);       // usa PI diretamente
    }
}
```

# Regras e Cuidados

1. O comando `package` **deve ser a primeira linha do arquivo** (antes de qualquer `import` ou código).

```java
package br.com.user.projeto;
import java.util.Scanner;
```

2. O nome do pacote **deve corresponder à estrutura de pastas** onde o arquivo está.
	- **Exemplo:**
		- Arquivo: `src/br/com/user/projeto/Main.java`  
		- Primeira linha: `package br.com.user.projeto;`

3. É possível ter vários  `import`, mas **apenas um `package`** por arquivo.

4. Os nomes de pacotes devem ser **letras minúsculas**, evitando conflito com classes (ex.: `java.util`, não `Java.Util`).

5. O **Java usa o nome do pacote para localizar classes compiladas** (`.class`), então mover um arquivo sem atualizar o `package` causa erro de compilação.

---
# Boas Práticas

- Use **nomes de pacotes únicos:** geralmente começa com o domínio da empresa invertido:

```text
br.com.nomeempresa.projeto.modulo
```

- Evite o uso de `impor *` em projetos grandes (pode causar conflitos).
- Mantenha uma **estrutura organizada**, por exemplo:

```java
modelo/  → classes de modelo (POJO)
controle/ → classes de controle ou lógica
visao/ → interface gráfica ou terminal
```

# Exemplo Prático

```swift
src/
 ├── br/com/user/app/Main.java
 ├── br/com/user/modelo/Pessoa.java
 └── br/com/user/modelo/Aluno.java
```

## `Pessoa.java`

```java
package br.com.user.modelo;

public class Pessoa {
    protected String nome;
    protected int idade;

    public Pessoa(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }

    public void exibir() {
        System.out.println("Nome: " + nome + " | Idade: " + idade);
    }
}
```

## `Aluno.java`

```java
package br.com.user.modelo;

public class Aluno extends Pessoa {
    private String curso;

    public Aluno(String nome, int idade, String curso) {
        super(nome, idade);
        this.curso = curso;
    }

    @Override
    public void exibir() {
        super.exibir();
        System.out.println("Curso: " + curso);
    }
}
```

## `Main.java`

```java
package br.com.user.app;

import br.com.tiago.modelo.Aluno; // importa do outro pacote

public class Main {
    public static void main(String[] args) {
        Aluno a = new Aluno("Maria", 23, "Engenharia de Software");
        a.exibir();
    }
}
```

**Saída:**

```txt
Nome: Maria | Idade: 23
Curso: Engenharia de Software
```

# Conclusão

| Conceito        | Descrição                                | Exemplo                             |
| --------------- | ---------------------------------------- | ----------------------------------- |
| `package`       | Define o pacote (pasta lógica) da classe | `package br.com.user.app;`          |
| `import`        | Permite usar classes de outros pacotes   | `import br.com.user.modelo.Pessoa;` |
| `import *`      | Importa todas as classes de um pacote    | `import java.util.*;`               |
| `import static` | Importa membros estáticos                | `import static java.lang.Math.*;`   |
