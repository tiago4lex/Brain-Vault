# Introdução

## O que é Encapsulamento?

**Encapsulamento** é o princípio da **ocultação de dados** (ou *"information hiding"*). Ele tem como objetivo **proteger os atributos e controlar o acesso** a eles, evitando que partes externas do programa modifiquem valores de forma incorreta.

Encapsular é colocar os atributos dentro de uma "caixa" e só permitir acesso a eles de maneira controlada, por meio de métodos.
 
>[!quote] Em outras palavras:
>"A diretriz de encapsulamento (ocultar informações de implementação de visualização) sugere que somente as informações sobre o que uma classe pode fazer devem ser visíveis externamente, não como ela é". *(BARNES; KÖLLING; FURMANKIEWICZ, 2022, p. 188).*

---
# Por que usar Encapsulamento?

Imagine que você tenha uma classe `contaBancaria`.

Seria perigoso deixar o saldo acessível diretamente, pois qualquer parte do código poderia alterar o valor sem controle.

Exemplo sem encapsulamento:

```java
public class ContaBancaria {
	double saldo;
}

public class Main {
	public static void main(String[] args) {
		ContaBancaria conta = new ContaBancaria();
		conta.saldo = -500;
		System.out.println("Saldo: " + conta.saldo);
	}
}
```

> **Problema:** o saldo foi alterado diretamente em `Main`, sem validação - isso quebra a segurança dos dados.

---
# Como aplicar o Encapsulamento

- Para privar um atributo ou método usamos a palavra `private`, para ela poder ser usada fora do pacote vamos declarar como pública para usar internamente não precisamos chamar ela como pública e new como privada.
- **Usar métodos públicos** chamados ***getters* e *setters*** para acessar e modificar esses atributos.

```java
private String nome;
private double peso;
private double preco;
private int quantidade;
private int estoque_maximo;
public double marge_lucro;
```

- Os métodos também pode ser encapsulados e protegidos para uso dentro da classe.

```java
private double valo_total(){
	return preco * quantidade;
}

public double lucro_total(){
	double valor, lucro;
	valor= this.valor_total();
	lucro = valor * marge_lucro;
	return lucro;
}
```

# Exemplo de Encapsulamento

```java
public class ContaBancaria {
	// Atributos privados (acesso restrito)
	private String titular;
	private double saldo;
	
	// Médodo getter (pega o valor)
	public String getTitular() {
		return titular;
	}
	
	// Método setter (define valor)
	public void setTitular(String titular) {
		this.titular = titular;
	}
	
	public double getSaldo() {
		return saldo;
	}
	
	public void depositar(double valor) {
		if (valor > 0) {
			saldo += valor;
			System.out.println("Depósito de R$" + valor + " realizado com sucesso!");
		} else {
			System.out.println("Valor de depósito inválido!");
		}
	}
	
	public void sacar(double valor) {
		if (valor > 0 && valor <= saldo) {
			saldo -= valor;
			System.out.println("Saque de R$" + valor + " realizado com sucesso!");
		} else {
			System.out.println("Saldo insuficiente ou valor inválido!");
		}
	}
}
```

# Usando a class com encapsulamento

```java
public class Main {
	public static void main(String[] args) {
		ContaBancaria conta = new ContaBancaria();
		
		// Definindo o titular
		conta.setTitular("Maria");
		
		// Tentando acessar diretamente (não é permitido!)
		// conta.saldo = 1000; ❌ ERRO: 'saldo' tem acesso privado
		
		// Usando métodos para alterar valores corretamente
		conta.depositar(500);
		conta.sacar(200);
		
		// Exibindo informações
		System.out.println("Titular: " + conta.getTitular());
		System.out.println("Saldo final: R$" + conta.getSaldo());
	}
}
```

**Saída:**

```text
Depósito de R$500.0 realizado com sucesso!
Saque de R$200.0 realizado com sucesso!
Titular: Maria
Saldo final: R$300.0
```

# Explicando os modificadores de acesso

| **Modificador**     | **Acesso dentro da classe** | **Mesmo pacote** | **Subclasses** | **Outros pacotes** |
| ------------------- | --------------------------- | ---------------- | -------------- | ------------------ |
| `public`            | ✅ Sim                       | ✅ Sim            | ✅ Sim          | ✅ Sim              |
| `protected`         | ✅ Sim                       | ✅ Sim            | ✅ Sim          | ❌ Não              |
| _(sem modificador)_ | ✅ Sim                       | ✅ Sim            | ❌ Não          | ❌ Não              |
| `private`           | ✅ Sim                       | ❌ Não            | ❌ Não          | ❌ Não              |

> **Encapsulamento usa o modificador** `private` para esconde os dados da classe.

# Getters e Setters

Os métodos `get` e `set` seguem uma convenção importante:

- **Getter:** retora o valor do atributo.

```java
public Tipo getNomeDoAtributo() { return atributo; }
```

- **Setter:** define o valor do atributo.

```java
public void setNomeDoAtributo(Tipo valor) { this.atributo = valor; }
```

- O prefixo `this` serve para **distinguir o atributo da variável local.**

**Exemplo:**

```java
public void setTitular(String titular) {
	this.titular = titular; // "this.titular" é o atributo da classe
}
```

## Exemplo Prático: Classe `Aluno` com Encapsulamento

```java
public class Aluno {
    private String nome;
    private double nota;

    // Getter e Setter
    public String getNome() {
        return nome;
    }

    public void setNome(String nome) {
        this.nome = nome;
    }

    public double getNota() {
        return nota;
    }

    public void setNota(double nota) {
        if (nota >= 0 && nota <= 10) {
            this.nota = nota;
        } else {
            System.out.println("Nota inválida! Digite um valor entre 0 e 10.");
        }
    }

    public boolean aprovado() {
        return nota >= 6.0;
    }
}

public class Main {
    public static void main(String[] args) {
        Aluno aluno = new Aluno();
        aluno.setNome("Maria");
        aluno.setNota(8.5);

        System.out.println("Aluno: " + aluno.getNome());
        System.out.println("Aprovado? " + aluno.aprovado());
    }
}
```

**Saída:**

```text
Aluno: Maria
Aprovado? true
```

# Benefícios do Encapsulamento

✅ **Segurança:** impede acesso direto a dados sensíveis.  
✅ **Controle:** garante que apenas valores válidos sejam atribuídos.  
✅ **Flexibilidade:** você pode mudar a implementação interna sem afetar o restante do código.  
✅ **Organização:** separa o que é interno da classe e o que pode ser usado fora dela.

## Quando usar

- Sempre que quiser **proteger atributos importantes** (como senhas, saldos, notas).    
- Quando precisar de **validação** antes de alterar um valor. 
- Quando quiser **evitar acesso direto** a variáveis internas da classe.
