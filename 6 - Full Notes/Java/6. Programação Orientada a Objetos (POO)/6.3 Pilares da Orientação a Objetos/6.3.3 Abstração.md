# Métodos Abstratos e Classes Abstratas

## O que são?

Uma **classe abstrata** é uma classe que **não pode ser instanciada diretamente**, ou seja, não pode criar objetos a partir dela.

Ela seve como **modelo** para outras classes herdarem.

Um **método abstrato** é um método **sem corpo** (sem implementação), que deve ser sobrescrito pelas classes filhas.

## Exemplo

```java
abstract class Funcionario {
	String nome;
	double salario;
	
	// Método abstrato → sem corpo
	abstract double calcularBonus();
	
	// Método concreto
	void exibirDados() {
		System.out.println("Nome: " + nome + ", Salário: " + salario);
	}
}

class Gerente extends Funcionario {
	@Override
	double calcularBonus() {
		return salario * 0.2;
	}
}

class Vendedor extends Funcionario {
	@Override
	double calcularBonus() {
		return salario * 0.1;
	}
}
```

```java
public class Main {
	public static void main(String[] args) {
		Funcionario f1 = new Gerente();
		f1.nome = "Maria";
		f1.salario = 5000;
		
		Funcionario f2 = new Vendedor();
		f2.nome = "Ana";
		f2.salario = 4000;
		
		System.out.prinln(f1.nome + " bônus: " + f1.calcularBonus());
		System.out.prinln(f2.nome + " bônus: " + f2.calcularBonus());
	} 
}
```

**Saída:**

```text
Maria bônus: 1000.0
Ana bônus: 400.0
```

## Quando usar classes e métodos abstratos

- Quando quer **forçar as subclasses a implementar um comportamento específico**.
- Quando quer criar uma **estrutura base**, mas que **não faz sentido ser instanciada diretamente**.
