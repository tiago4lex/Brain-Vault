# O que é Herança?

A herança é um dos pilares fundamentais da programação Orientada a Objetos (POO), junto com **encapsulamento, polimorfismo e abastração**.

Ela permite que uma classe **(filha ou subclasse)** herde atributos e métodos de outras classe (pai ou superclasse), **reaproveitando código** e promovendo a reutilização e organização.

Em outras palavras: **herança é mecanismo que permite criar uma nova classe baseada em uma classe existente**.

---
# Exemplo conceitual (sem código)

Imagine uma hierarquia de seres vivos:

- Um **Animal** tem características como: `nome`, `idade`, e comportamentos como `comer()` e `dormir()`.
- Um **Cachorro** também é um animal, mas possui características adicionais, como `latir()`.

Nesse caso:

- **Animal** é a super classe (classe mãe).
- **Cachorro** é a subclasse (classe filha).
- O **Cachorro** herda de **Animal** tudo que um animal já tem e ainda adiciona o que for exclusivo dos cachorros.

---
# Sintaxe de Herança em Java

A  herança é declarada com a palavra-chave `extends`:

```java
class Subclasse extends Superclasse {
	// novos atributos e métodos a serem adicionados
}
```

## Exemplo básico

```java
// Classe Pai (superclasse)
public class Aimal {
	String nome;
	
	public void comer() {
		System.out.prinln(nome + " está comendo...");
	}
}

// Classe Filha (subclasse)
public class Cachorro extends Animal {
	public void latir() {
		System.out.println(nome + " está latindo...");
	}
}

// Programa Principal
public class Main {
	public static void main(String[] args) {
		Cachorro c = new Cachorro();
		c.nome = "Rex";
		c.comer();  // herdado da classe Animal
		c.latir();  // herdado da própria classe
	}
}
```

**Saída:**

```text
Rex está comendo...
Rex está latindo...
```

---
# Como funciona a Herança

- A **subclasse** herda **todos os atributos e métodos públicos e protegidos** da superclasse.
- Pode **acessar diretamente** esses membros herdados (se forem `public` ou `protected`).
- Pode **adicionar novos atributos e métodos próprios**.
- Pode **redefinir (sobrescrever)** métodos da classe pai, isso se chame **polimorfismo**.
- A herança cria uma **relação "é um" *(IS-A)***:
	- Um `Cachorro` é um `Animal`.
	- Mas nem todo `Animal` é um `Cachorro`.

---
# Hierarquia de Classes

Java **permite herança simples**, ou seja, **cada classe só pode herdar de uma superclasse direta.**

**Exemplo de Hierarquia:**

```
Animal
├── Cachorro
├── Gato
└── Cavalo
```

Cada subclasse herda de `Animal`, e pode ainda ter suas próprias subclasses.

---
# A palavra-chave `super`

A palavra `super` é usada para:

1. **Chamar o construtor da superclasse**.
2. **Acessar o método ou atributos da superclasse**.

**Exemplo:**

```java
public class Animal {
	String nome;
	
	public Animal(String nome) {
		this.nome = nome;
	}
	
	public void comer() {
		System.out.println(nome + " está comendo...");
	}
}

public class Cachorro extends Animal {
	public Cachorro(String nome) {
		super(nome); // chama o construtor da superclasse
	}
	
	@Override
	public void comer() {
		super.comer(); // chama o método da superclasse
		System.out.printl(nome + " comeu a ração de cachorro.");
	}
}
```

# Herança e modificadores de Acesso

| Modificador                 | Visível na Subclasse?                | **Observação**                                              |
| --------------------------- | ------------------------------------ | ----------------------------------------------------------- |
| `public`                    | ✅ Sim                                | Pode ser acessado de qualquer lugar.                        |
| `protected`                 | ✅ Sim                                | Pode ser acessado pelas subclasses (mesmo em outro pacote). |
| `default` (sem modificador) | ⚠️ Apenas se estiver no mesmo pacote |                                                             |
| `private`                   | ❌ Não                                | Não é herdado diretamente.                                  |

**Exemplo:**

```java
public class Animal {
	private int idade;
	protected String nome;
	
	public void setIdade(int idade) {
		this.idade = idade;
	}
	
	public int getIdade() {
		return idade;
	}
}
```

Aqui:

- `nome` pode ser acessado diretamente por subclasses.
- `idade` é privado → só é acessível via getters/setters.

---
# Sobrescrita de métodos (`@Override`)

Uma subclasse pode **redefinir o comportamento** de um método herdade da superclasse usando a **anotação** `@Override`.

```java
public class Animal {
	public void emitirSom() {
		System.out.prinln("O animal fez um som.");
	}
}

public class Gato extends Animal {
	@Override
	public void emitirSom() {
		System.out.println("O gato miou!");
	}
}
```

---
# Métodos `final` e classes `final`

- `final`**em método:** impede que o método seja sobrescrito.
- `final` **em classe:** impede que a classe seja herdade.

**Exemplo:**

```java
public final class Animal {
	// nenhuma classe pode herdar de Animal
}

public class Cachorro extends Animal { } // ERRO de compilação
```

---
# Herança e Construtores

- O construtor da superclasse não é herdado, mas pode ser chamado com `super()`.
- Se a superclasse tiver **apenas construtor com parâmetros**, a subclasse deve chamar `super(parâmetros)`.

**Exemplo:**

```java
public class Pessoa {
	String nome;
	
	public Pessoa(String nome) {
		this.nome = nome;
	}
}

public class Aluno extends Pessoa {
	int matricula;
	
	public Aluno(String nome, int matricula) {
		super(nome); // obrigatório, pois Pessoa não tem construtor padrão
		this.matricula = matricula;
	}
}
```

---
# Quando usar Herança

Herança deve ser usado quando há uma **relação natural de generalização/especialização** ("é um tipo de").

### ✅ Exemplos adequados

- `Funcionário` → `Gerente`, `Programador`, `Estagiario`
- `Veiculo` → `Carro`, `Moto`, `Bicicleta`
- `FormaGeometrica` → `Quadrado`, `Circulo`, `Triangulo`

### ❌ Evitar herança quando

- Não há uma relação "é um".
- A subclasse só está herdando para **reaproveitar código**.
- A herança gera dependência desnecessária entre classes.

---
# Herança vs Composição

Nem sempre herança é a melhor escolha.
Quando a relação for **"tem um"** *(HAS-A)*, é necessário usar **composição** (atributo da classe).

| Situação | Exemplo            | Recomendação |
| -------- | ------------------ | ------------ |
| "é um"   | Aluno é uma Pessoa | Herança      |
| "tem um" | Carro tem um Motor | Composição   |

**Exemplo (composição):**

```java
public class Motor {
	public void ligar() {
		System.out.println("Motor ligado!");
	}
}

public class Carro {
	private Motor motor = new Motor(); // composição
	
	public void ligar() {
		motor.ligar();
		System.out.println("Carro em movimento!");
	}
}
```

# Resumo

| Conceito         | Descrição                                 | Exemplo                              |
| ---------------- | ----------------------------------------- | ------------------------------------ |
| `extends`        | Indica herança de uma classe              | `class Aluno extends Pessoa`         |
| `super`          | Acessa construtor/métodos da classe mãe   | `super(nome)`                        |
| `@Override`      | Indica sobrescrita de método              | `@Override public void apresentar()` |
| Herança simples  | Java permite herdar apenas uma classe     | `class B extends A`                  |
| Herança múltipla | Não suportada (use interfaces)            | ❌                                    |
| Classe abstrata  | Não pode ser instanciada, serve de modelo | `abstract class Forma {}`            |
