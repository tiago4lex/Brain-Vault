# Polimorfismo

## O que é?

A palavra *polimorfismo* vem do grego e significa **"muitas formas"**.
Na Programação Orientada a Objetos, o **polimorfismo** permite que um **mesmo método tenha comportamentos diferentes** dependendo do objeto que o utiliza.

Isso torna o código mais flexível, reutilizável e fácil de manter.

## Tipos de Polimorfismo em Java

### 1. Polimorfismo de Sobrescrita (Overriding)

- Um método de uma **classe filha substitui** (sobrescreve) um método com o **mesmo nome e assinatura da classe pai**.

### 2. Polimorfismo de Sobrecarga (Overloading)

- Um método tem o **mesmo nome, mas assinaturas diferentes** (quantidades ou tipo de parâmetros diferentes) **dentro de uma mesma classe**.

## Sobrescrita (Override)

```java
class Animal {
	void emitirSom() {
		System.out.println("O animal faz um som.");
	}
}

class Cachorro extends Animal {
	@Override
	void emitirSom() {
		System.out.println("O cachorro está latindo!");
	}
}

class Gato extends Animal {
	void emitirSom() {
		System.out.prinln("O gato está miando!");
	}
}

public class Main {
	public static void main(String[] args) {
		Animal a1 = new Cachorro();
		Animal a2 = new Gato();
		
		a1.emitirSom(); //  O cachorro está latindo!
		a2.emitirSom(); // O gato está miando!
	}
}
```

Mesmo que as variáveis sejam do tipo `Animal`, o Java **chama o método correspondente à classe real do objeto**.

## Sobrecarga (Overloading)

```java
class Calculadora {
	int somar(int a, int b) {
		return a + b;
	}
	
	double somar(double a, double b) {
		return a + b;
	}
	
	int somar(int a, int b, int c) {
		return a + b + c;
	}
}
```

O método `somar()` foi declarado **três vezes com assinaturas diferentes** (quantidade e tipo de parâmetros).

O Java decide automaticamente qual versão usar com base nos argumentos passados.

## Quando usar Polimorfismo

- Quando há heranças e você quer **diferenciar comportamentos** entre classes filhas.
- Quando deseja **usar uma referência genérica (classe pai)** para **manipular vários tipos de objetos**.
