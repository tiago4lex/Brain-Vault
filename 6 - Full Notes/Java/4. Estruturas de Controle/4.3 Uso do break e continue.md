# Introdução

No **Java**, dentro de estruturas de repetição (`for`, `while`, `do-while`) e também em instruções `switch`, temos duas palavras-chave muito importantes para **controlar o fluxo da execução**:

- `break`
- `continue`

Eles permitem **interromper** ou **pular** partes de um loop, tornando o código mais eficiente e flexível.

---
# O comando `break`

O `break` serve para **interromper imediatemente a execução** do laço ou do bloco de código que ele está.

Quando o Java encontra um `break`, ele **sai do loop** e continua a execução do programa a partir da primeira linha após o laço.

## Exemplo 1 - `break` em um `for`

```java
public class ExemploBreak {
	public static void main(String[] args) {
		for (int i = 1; i <= 10; i++) {
			if (i == 5) {
				System.out.println("Parando o loop no número " + i);
				break; // Interrompe o laço quando i == 5
			}
			System.out.println("Número: " + i);
		}
		System.out.println("Fim do programa.");
	}
}
```

**Saída:**

```text
Número: 1
Número: 2
Número: 3
Número: 4
Parando o loop no número 5
Fim do programa.
```

> Perceba que o laço **não chega até o 10**, ele é interrompido no número 5.

## Exemplo 2 - `break` em um `while`

```java
public class ExemploBreakWhile {
	public static void main(String[] args) {
		int contador = 1;
		
		while (contador <= 10) {
			if (contador == 7) {
				System.out.println("Interrompendo no " + contador);
				break;
			}
			System.out.prinln("Contador: " + contador);
			contador++;
		}
	}
}
```

> Aqui, o loop roda até que `contador` seja 7, depois o `break` interrompe a execução.

# Comando `continue`

O `continue` serve para **pular a iteração atual e voltar para o início do laço**, verificando novamente a condição.

Ou seja, **ele não interrompe o loop inteiro**, apenas **ignora uma repetição específica**.

## Exemplo 1 - `Continue` em um `for`

```java
public class ExemploContinue {
	public static void main(String[] args) {
		for (int i = 1; i <= 10; i++) {
			if (i % 2 == 0) {
				continue; // pula os números pares
			}
			System.out.prinln("Número ímpar: " + i);
		}
	}
}
```

**Saída:**

```text
Número ímpar: 1
Número ímpar: 3
Número ímpar: 5
Número ímpar: 7
Número ímpar: 9
```

> É possível perceber que o `continue` **pula os pares**, mas o laço continua até o fim.

## Exemplo 2 - `continue` em um `while`

```java
public class ExemploContinueWhile {
	public static void main(String[] args) {
		int i = 0;
		
		while (i < 10) {
			i++;
			if (i == 5) {
				continue; // pula o número 5
			}
			System.out.println("Número: " + i);
		}
	}
}
```

**Saída:**

```text
Número: 1
Número: 2
Número: 3
Número: 4
Número: 6
Número: 7
Número: 8
Número: 9
Número: 10
```

> O número `5` é **ignorado** por causa do `continue`.

---
# Diferença entre `break` e `continue`

|**Comando**|**Efeito**|
|---|---|
|`break`|Sai **completamente do laço**. Nenhuma iteração adicional será executada.|
|`continue`|Pula a **iteração atual** e vai para a próxima, sem sair do laço.|
## Usos comuns

- `break`:
	- Quando queremos **interromper a execução ao encontrar uma condição**.
	- Usado também em instruções `switch`.

- `continue`:
	- Quando precisamos **ignorar certas condições** dentro de um loop, mas **continua rodando o restante**.

# Exemplo prático: Encontrar um número em uma lista

```java
public class BuscarNumero {
	public static void main(String[] args) {
		int[] numeros = {1, 4, 7, 10, 15, 20};
		
		for (int n : numeros) {
			if (n == 10) {
				System.out.println("Número encontrado: " + n);
				break; // encerra o loop ao encontrar
			}
		}
	}
}
```

> Esse código busca o número `10` na lista e interrompe a busca assim que encontra.

---
# Exemplo prático: Imprimir apenas números maiores que 5

```java
public class IgnorarNumeros {
	public static void main(String[] args) {
		int[] numero = {2, 4, 6, 8, 10};
		
		for (int n : numeros) {
			if (n <= 5) {
				continue; // ignora números menores ou iguais a 5
			}
			System.out.println("Número válido: " + n);
		}
	}
}
```

**Saída:**

```text
Número válido: 6
Número válido: 8
Número válido: 10
```

---
# Conclusão

- `break`: sai **completamente** do laço.
- `continue`: **pula a iteração atual**, mas o loop continua.
- São ferramentas úteis para **controlar loops dinamicamente**.
