# Introdução

## O que é a Conversão de Valores?

Em Java, conversão de valores é o processo de **transformar um valor de um tipo de dado em outro tipo**. Isso pode ocorrer de forma **implícita (automática)** ou **explícita(manual, com casting)**.

Esse processo é necessário quando trabalhamos com tipos de dados diferentes, para garantir que os valores sejam compatíveis durante operações ou atribuições.

---
# Tipos de Conversão

## Conversão Implícita *(Widening Casting)*

- Também chamada de **promoção automática de tipo**.
- Acontece quando convertemos um tipo **menor para um tipo maior** (mais abrangente).
- Não há perda de dados e o Java Compiler faz automaticamente.

**Ordem dos tipos numéricos do menor para o maior:**

`byte → short → int → long → float → double`

**Exemplo:**

```java
public class ConversaoImplicita {
	public static void main(String[] args) {
		int numInteiro = 10;
		double numDecimal = numInteiro; // int convertido automaticamente para double
		
		System.out.println(numDecimal); // Saída: 10.0
	}
}
```

## Conversão Explícita *(Narrowing Casting)*

- Também chamada de **casting manual**.
- É usada quando convertemos um tipo **maior para um tipo menor**.
- Pode ocorrer **perda de dados** (valores ou precisão).
- Precisa do operador de casting `(tipo)`.

**Exemplo:**

```java
public class ConversaoExplicita {
	public static void main(String[] args) {
		double numDecimal = 9.78;
		int numInteiro = (int) numDecimal // casting manual de double para int
		
		System.out.println(numInteiro);  // Saída: 9 (a parte decimal é perdida)
	}
}
```

# Conversão entre tipos de dados Não Numéricos

- De `String` para tipos numéricos
	- É usado o método da classe *Integer, Double, etc.*

```java
public class StringParaNumero {
	public static void main(String[] args) {
		String texto = "123";
		int numero = Integer.parseInt(texto);
		double numeroDecimal = Double.parseDouble("45.6");
		
		System.out.println(numero); // 123
		System.out.println(numeroDecimal); // 45.6
	}
}
```

- De tipos numéricos para `String`:
	- É usado `String.valuesOf()` ou concatenação com `""`.

```java
int numero = 10;
String texto1 = String.valueOf(numero);
String texto2 = numero + "";
```

# Tabela de Casting Numérico

| Tipo menor | Pode ser convertido para...               | Tipo maior       | Casting necessário |
| ---------- | ----------------------------------------- | ---------------- | ------------------ |
| `byte`     | `short`, `int`, `long`, `float`, `double` | automático       | Não                |
| `short`    | `int`, `long`, `float`, `double`          | automático       | Não                |
| `char`     | `int`, `long`, `float`, `double`          | automático       | Não                |
| `int`      | `long`, `float`, `double`                 | automático       | Não                |
| `long`     | `float`, `double`                         | automático       | Não                |
| `float`    | `double`                                  | automático       | Não                |
| `double`   | `float`, `long`, `int`, `short`, `byte`   | manual (casting) | Sim                |
| `float`    | `long`, `int`, `short`, `byte`            | manual (casting) | Sim                |
| `long`     | `int`, `short`, `byte`                    | manual (casting) | Sim                |
| `int`      | `short`, `byte`                           | manual (casting) | Sim                |

# Cuidados com Casting

- Pode haver **perda de precisão** ao converter tipos decimais para inteiros.
- Pode ocorrer **estouro de valores** ao converter tipos grandes para menores.
- Sempre verifique se o valor cabe no tipo de destino antes de converter.

**Exemplo de estouro:**

```java
public class EstouroCasting {
	public static void main(String[] args) {
		int valor = 130;
		byte pequno = (byte) valor; // byte vai de -128 a 127
		
		System.out.println(pequeno); // Saída: -126 (valor "estorou")
	}
}
```

---
# Resumo

- **Conversão implícita**: de tipo menor para maior, automática e segura.
- **Conversão explícita (casting)**: de tipo maior para menor, manual e pode perder dados.
- Usar métodos utilitários para conversão entre `String` e tipos numéricos.
