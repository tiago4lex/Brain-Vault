# O que é um Objeto?

Em **JavaScript**, um objeto é uma coleção de pares de chave e valor (também chamados de propriedades).

Cada propriedade possui:

- **Uma chave *(key)***: geralmente uma string ou símbolo.
- **Um valor *(value)***: que pode ser de qualquer tipo (número, string, função, outro objeto, etc.).

**Exemplo básico:**

```js
const pessoa = {
	nome: "Maria",
	idade: 23,
	profissao: "Estudante de Engenharia de Software"
};
```

Aqui, `pessoa` é um **objeto**, e ele possui três propriedades:

- `nome` → `"Maria"`
- `idade` → `23`
- `profissao` → `"Estudante de Engeharia de Software"`

---
# Acessando Propriedades

Existem **duas formas principais** de acessar propriedades em objetos no JavaScript:

## a) Notação de Ponto *(dot notation)*

A forma mais comum e legível.
Sintaxe:

```js
objeto.propriedade
```

**Exemplo:**

```js
console.log(pessoa.nome);      // "Maria"
console.log(pessoa.idade);     // 23
console.log(pessoa.profissao); // "Estudante de Engenharia de Software"
```

Quando usar:

- Quando o nome da propriedade é **válido como um identificador JavaScript** (sem espaços, acentos, número no início, etc.).

## b) Notação de Colchetes *(bracket notation)*

Permite acessar propriedades usando **string** entre colchetes (`[]`).
Sintaxe:

```js
objeto["propriedade"]
```

**Exemplo:**

```js
console.log(pessoa["nome"]);  // "Maria"
console.log(pessoa["idade"]); // 23
```

Quando usar:

- Quando o nome da propriedade contém caracteres especiais ou espaços.
- Quando o nome da propriedade está armazenada em uma variável.

**Exemplo com variável:**

```js
const chave = "profissao";
console.log(pessoa[chave]); // "Estudante de Engenharia de Software"
```

**Exemplo com nomes inválidos para notação de ponto:**

```js
const carro = {
	"marca do carro": "Toyota",
	"ano-fabricacao": 2020
};

console.log(carro["marca do carro"]); // Correto
console.log(carro["ano-fabricacao"]); // Correto
```

>[!warning] Atenção:
>`carro.marca do carro` daria erro de sintaxe.

# Acessando Propriedades Aninhadas

Objetos podem conter **outros objetos dentro deles**.

Nesse caso, basta encadear os acessos com o operador `.` ou `[]`.

**Exemplo:**

```js
const usuario = {
	nome: "Maria",
	endereco: {
		cidade: "Curitiba",
		estado: "PR",
		cep: "80000-000"
	}
};

console.log(usuario.endereco.cidade);    // Curitiba
console.log(usuario["endereco"]["cep"]); // 80000-000
```

---
# Acessando Propriedades com Valor de Função (Métodos)

Quando o valor de uma propriedade é uma **função**, ela é chamada de **método**.

**Exemplo:**

```js
const aluno = {
	nome: "Maria",
	saudar: function() {
		console.log("Olá, meu nome é " + this.nome);
	}
};

aluno.saudar(); // "Olár, meu nome é Maria"
```

> O `this` dentro de um objeto **referencia o próprio objeto** (no caso, `aluno`).

---
# Acessando Propriedades Dinamicamente

É possível **acessar propriedades com base em variáveis** ou construir nomes dinamicamente.

**Exemplo:**

```js
const produto = {
	nome: "Notebook",
	preco: 4500,
	estoque: 10
};

function mostrarPropriedade(obj, chave) {
	console.log(obj[chave]);
}

mostrarPropriedade(produto, "preco"); // 4500
```

Aqui, `chave` é uma variável, e o uso de `obj[chave]` permite acessar dinamicamente a propriedade desejada.

# Verificando se uma Propriedade Existe

Você pode verificar se uma propriedade existe em um objeto de três formas principais:

## a) Operador `in`

```js
console.log("nome" in pessoa);   // true
console.log("altura" in pessoa); // false
```

## b) Método `hasOwnProperty()`

Verifica se a propriedade **pertence diretamente** ao objeto (e não ao seu protótipo).

```js
pessoa.hasOwnProperty("idade"); // true
```

## c) Comparando com `undefined`

```js
if (pessoa.nome !== undefined) {
	console.log("Propriedade existe!");
}
```

# Acesso Seguro com Encadeamento Opcional (`?.`)

Introduzido no **ES2020**, o operador `?.` evita erros quando tentamos acessar propriedades de objetos **que podem ser `undefined` ou `null`**.

**Exemplo:**

```js
const cliente = {
	nome: "Ana",
	endereco: {
		rua: "Rua das Flores"
	}
};

console.log(cliente.endereco?.rua);    // "Rua das Flores"
console.log(cliente.telefone?.numero); // undefined (sem erro!)
```

Sem o `?.` o segundo `console.log` causaria erro:

```js
TypeError: Cannot read properties of undefined
```

# Listando Todas as Propriedades de um Objeto

O JavaScript fornece métodos para listar propriedades.

**Exemplo:**

```js
const carro = {
  marca: "Toyota",
  modelo: "Corolla",
  ano: 2020
};

console.log(Object.keys(carro));   // ["marca", "modelo", "ano"]
console.log(Object.values(carro)); // ["Toyota", "Corolla", 2020]
console.log(Object.entries(carro)); 
// [["marca", "Toyota"], ["modelo", "Corolla"], ["ano", 2020]]
```

Esse métodos são úteis para **iterar** sobre propriedades.

---
# Iterando sobre as Propriedades de um Objeto

O laço `for...in` é usado para percorrer todas as **chaves** de um objeto.

**Exemplo:**

```js
for (let chave in carro) {
	console.log(chave + ": " + carro[chave]);
}
```

**Saída:**

```text
marca: Toyota
modelo: Corolla
ano: 2020
```

# Resumo Geral

|Ação|Sintaxe|Descrição|
|---|---|---|
|Acesso direto|`obj.propriedade`|Acesso comum e legível|
|Acesso dinâmico|`obj["propriedade"]`|Quando a chave é variável|
|Acesso aninhado|`obj.prop1.prop2`|Acessa objetos dentro de objetos|
|Verificação|`"chave" in obj`|Verifica existência da propriedade|
|Acesso seguro|`obj?.prop`|Evita erro caso o objeto seja `undefined`|

---
# Boas Práticas

✅ Prefira **notação de ponto** sempre que possível (mais legível).  
✅ Use **notação de colchetes** para chaves dinâmicas ou nomes inválidos.  
✅ Utilize o operador `?.` para evitar erros de propriedades indefinidas.  
✅ Evite acessar propriedades sem verificar se o objeto realmente existe.  
✅ Lembre-se: **funções também são valores**, e podem ser propriedades!
