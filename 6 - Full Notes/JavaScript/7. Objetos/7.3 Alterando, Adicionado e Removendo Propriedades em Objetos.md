# Alterando Propriedades de um Objeto

Para alterar o valor de uma propriedade existente, basta **acessar a propriedade e atribuir um novo valor**.

### Exemplo 1 - Alteração de propriedade existente

```js
const usuario = {
  nome: "Maria",
  idade: 23,
  ativo: true
};

// Alterando o valor de uma propriedade
usuario.idade = 24;
usuario.ativo = false;

console.log(usuario);
```

**Saída:**

```text
{ nome: 'Maria', idade: 24, ativo: false }
```

---
# Adicionado Novas Propriedades

Podemos adicionar uma nova propriedade a um objeto existente usando a mesma notação (`objeto.propriedade = valor`) ou colchetes (`objeto["propriedade"] = valor`).

### Exemplo 2 - Adição de nova propriedade

```js
const usuario = {
  nome: "Maria",
  idade: 23
};

// Adicionando novas propriedades
usuario.email = "maria@example.com";
usuario["cidade"] = "Curitiba";

console.log(usuario);
```

**Saída:**

```text
{ nome: 'Maria', idade: 23, email: 'maria@example.com', cidade: 'Curitiba' }
```

**Explicação:**

- `usuario.email = "maria@example.com"` adiciona uma nova propriedade chamada `email`.
- `usuario["cidade"] = "Curitiba"` faz o mesmo, mas usando a notação de colchetes (útil quando o nome da propriedade é dinâmico ou contém espaços).

# Removendo Propriedades de um Objeto

Para remover uma propriedade, usamos o operador `delete`.

### Exemplo 3 - Removendo propriedades

```js
const usuario = {
  nome: "Maria",
  idade: 23,
  email: "maria@example.com"
};

// Removendo a propriedade 'email'
delete usuario.email;

console.log(usuario);
```

**Saída:**

```text
{ nome: 'Maria', idade: 23 }
```

**Explicação:**

- `delete usuario.email` remove completamente a propriedade `email` do objeto.

# Notação de Ponto vs Notação de Colchetes

| Tipo de Notação    | Exemplo                                                | Quando usar                                                                          |
| ------------------ | ------------------------------------------------------ | ------------------------------------------------------------------------------------ |
| **Ponto (.)**      | `usuario.nome`                                         | Quando o nome da propriedade é fixo e válido como identificador JavaScript.          |
| **Colchetes ([])** | `usuario["nome completo"]` ou `usuario[chaveDinamica]` | Quando o nome da propriedade contém espaços, caracteres especiais ou é uma variável. |

# Propriedades Dinâmicas

É possível criar propriedades com nomes **definidos em tempo de execução**, usando variáveis.

```js
const chave = "curso";
const aluno = {
  nome: "Maria"
};

// Adicionando propriedade com nome dinâmico
aluno[chave] = "JavaScript Avançado";

console.log(aluno);
```

**Saída:**

```text
{ nome: 'Maria', curso: 'JavaScript Avançado' }
```

**Explicação:**

- Como `chave` tem o valor `"curso"`, a linha `aluno[chave] = ...` cria uma propriedade chamada `curso`.

# Atualizando Várias Propriedades ao Mesmo Tempo

Você pode atualizar ou adicionar várias propriedades com o método `Object.assign()` ou o operador **spread (`...`)**.

### Exemplo 4 - Usando `Object.assign()`

```js
const usuario = { nome: "Maria", idade: 23 };

Object.assign(usuario, { idade: 24, cidade: "Curitiba" });

console.log(usuario);
```

**Saída:**

```text
{ nome: 'Maria', idade: 24, cidade: 'Curitiba' }
```

### Exemplo 5 - Usando spread(`...`)

```js
const usuario = { nome: "Maria", idade: 23 };
const novosDados = { idade: 24, cidade: "Curitiba" };

const usuarioAtualizado = { ...usuario, ...novosDados };

console.log(usuarioAtualizado);
```

**Saída:**

```text
{ nome: 'Maria', idade: 24, cidade: 'Curitiba' }
```

**Explicação:**

- O operador spread cria um **novo objeto**, copiando as propriedades de outros.
- Caso haja propriedades com o mesmo nome, o último valor sobrescreve o anterior.

# Cuidados ao Modificar Objetos

## 1. Objetos são mutáveis

Mesmo declarados com `const`, seu conteúdo pode mudar:

```js
const obj = {a: 1};
obj.a = 2; // permitido
```

## 2. Evite modificar objetos diretamente

Se estiver trabalhando com **imutabilidade** (como em React).
Use spread operator para criar **cópias** em vez de alterar o original.

## 3. Propriedades inexistentes

Propriedades inexistentes retornam `undefined`, e tentar acessá-las não gera erro:

```js
const carro = {};
console.log(carro.modelo); // undefined
```
