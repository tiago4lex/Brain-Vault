# Introdução

## O que é Conversão de Tipos?

Em JavaScript, **conversão de tipo** significa transformar um valor de um tipo de dado para outro, por exemplo:

- `Number` → `String`
- `String` → `Number`
- `Boolean` → `String`
	e assim por diante.

Existem dois tipos de conversão:

1. **Implícita** (coerção) → feita automaticamente pelo JavaScript
2. **Explícita** → feita manualmente pelo programador.

---
# Conversão de `Number` para `String`

## Explícita

Podemos usar:

- `String(valor)`
- `valor.toString()`
- Template literals ``${valor}``

```javascript
let numero = 123;

// Usando String()
let texto1 = String(numero)

// Usanto toString()
let texto2 = numero.toString();

// Usando template literal
let texto3 = `${numero}`;

console.log(typeof texto1); // String
console.log(typeof texto2); // String
console.log(typeof texto3); // String
```

---
# Conversão de `String` para `Number`

## Explícita

Podemos usar:

- `Number(valor)`
- `parseInt(valor)` → converte para inteiro
- `parseFloat(valor)` → converte para decimal

```javascript
let texto = "42.5";

// Usando Number()
let num1 = Number(texto);    // 42.5

// Usando parseInt()
let num2 = parseInt(texto);  // 42 (remove decimal)

// Usadno parseFloat()
let num3 = parseFloat(texto); // 42.5

console.log(num1, num2, num3);
```

## Cuidados

Se a string não for numérica, o resultado será `NaN` ***(Not a Number)***:

```javascript
let invalido = Number("abc");
console.log(invalido); // NaN
```

---
# Conversão para Boolean

- Valores **falsos** → `false`: `0`, `""` (string vazia), `null`, `undefined`, `NaN`
- Qualquer outro valor → `true`

```javascript
console.log(Boolean(0));        // false
console.log(Boolean(1));        // true
console.log(Boolean(""));       // false
console.log(Boolean("texto"));  // true
```

---
# Conversão Implícita (Coerção de Tipo)

JavaScript, em alguns casos, converte tipos **automaticamente**:

```javascript
console.log("5" + 3);   // "53" (número vira string)
console.log("5"-3);     // "2"  (string vira número)
console.log("10" * "2") // "20"
```

>[!warning] Atenção
>Isso pode causar bugs - prefira conversão **explícita**.

# Abreviação de Números Float

Às vezes precisamos limitar casas decimais ou formatar números.

## Usando `toFixed()`

- Retorna **string** com número arredondado

```js
let num = 3.14159265

console.log(num.toFixed(2));  // "3.14"
console.log(num.toFixed(4));  // "3.1416"
```

## Usando `toPrecision`

- Define o **número total de dígitos**.

```javascript
let num = 123.456

console.log(num.toPrecision(4));   // "123.5"
console.log(num.toPrecision(2));   // "1.2e+2"
```

## Convertendo para número novamente

- Se precisar de **número** ao invés de string:

```js
let abreviado = Number(num.toFixed(2));
console.log(abreviado);  // 3.14
```

## Formatação com `Intl.NumberFormat`

```js
let valor = 1234.567
let formatado = new Intl.NumberFormat("pt-BR", {
	minimumFractionDigits: 2,
	maximumFractionDigits: 2
}).format(valor);

console.log(formatado);  // "1.234,57"
```

| **Conversão**      | **Método(s)**                                | **Exemplo**                     |
| ------------------ | -------------------------------------------- | ------------------------------- |
| Number → String    | `String()`, `toString()`, template literal   | `String(10)` → `"10"`           |
| String → Number    | `Number()`, `parseInt()`, `parseFloat()`<br> | `parseFloat("3.14")` → `3.14`   |
| Qualquer → Boolean | `Boolean()`                                  | `Boolean("")` → `false`         |
| Float abreviado    | `toFixed()`, `toPrecision()`                 | `3.14159.toFixed(2)` → `"3.14"` |
