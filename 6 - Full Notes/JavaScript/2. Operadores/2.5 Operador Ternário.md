# Introdução

## O que é o Operador Ternário?

O **operador ternário** (também chamado de operador condicional) é uma forma **mais curta e expressiva** de escrever uma estrutura condicional `if...else` no JavaScript.

Ele recebe esse nome porque envolve **três operandos:**

```js
condição ? expressãoSeVerdadeira : expressãoSeFalsa
```

- **condição** → Uma expressão booleana que será avaliada.
- **expressãoSeVerdadeira** → O valor retornado caso a condição seja verdadeira.
- **expressãoSeFalsa** → O valor retornado caso a condição seja falsa.

---
# Sintaxe Básica

```js
let resultado = condição ? valor1: valor2;
```

**Exemplo simples:**

```js
let idade = 18;
let status = idade >= 18 ? "Maior de Idade" : "Menor de Idade";

console.log(status); // Saída: "Maior de Idade"
```

- Se `idade >= 18` for **true**, retorna `"Maior de Idade"`.
- Caso contrário, retorna `"Menor de Idade"`

---
# Comparando com `if...else`

Usando `if...else`

```js
let nota = 7;
let situacao;

if (nota >= 6){
	situaca = "Aprovado";
} else {
	situacao = "Reprovado";
}

console.log(situacao); // Aprovado
```

## Usando Operador Ternário

```js
let nota = 7;
let situacao = nota >= 6 ? "Aprovado" : "Reprovado";

console.log(situacao); // Aprovado
```

> A versão com operador ternário é **mais curta e direta**.


---
# Uso com Expressões Mais Complexas

```js
let numero = 10;
numero % 2 === 0
	? console.log("Número par")
	: console.log("Número impar");
```

---
# Aninhamento de Operadores Ternários

É possível **aninhar vários operadores ternários** , isso pode deixar o código **difícil de ler**.

```js
let nota = 8;
let resultado = nota >= 9 ? "Excelente"
				: nota >= 7 ? "Bom"
				: nota >= 5 ? "Regular"
				: "Reprovado";
				
console.log(resultado); // Bom
```

> [!attention] Cuidado:
> Quando o código começa a ficar muito complexo, é melhor usar `if...else` para mantar a legibilidade.


---
# Quando usar?

✅ Quando a decisão for **simples e curta**.  
❌ Evite em lógicas complexas, pois podem comprometer a clareza do código.

---
# Dicas

- Use parênteses para evitar ambiguidades em expressões complexas.
- Mantenha o ternário em **uma única linha curta** sempre que possível.
- Não use ternários apenas para efeitos colaterais (como `console.log`) sem atribuição de valor.

---
# Exemplo Final

```js
let usuario = {nome: "Maria", ativo: true};

let mensagem = usuario.ativo
	? `Bem-vindo, ${usuario.nome}`
	: "Usuário inativo";

console.log(mensagem); // Saída: "Bem-vindo, Maria"
```
