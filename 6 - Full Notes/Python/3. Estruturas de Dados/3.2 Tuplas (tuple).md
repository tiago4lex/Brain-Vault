# Introdução

Tuplas são estruturas de dados usadas para armazenar **coleçãoes ordenadas de valores** semelhantes às listas, porém **imutáveis**.

Isso significa que, após criadas, **não podem ser modificadas**.

Principais características:

- Ordenadas
- Imutáveis
- Aceitam tipos diferentes
- Permitem valores duplicados
- Mais seguras e eficientes em alguns cenários

---
# Criação de Tuplas

## 1. Criando uma tupla simples

```python
numeros = (1, 2, 3)
```

## 2. Tupla com tipos mistos

```python
dados = ("Maria", 23, True)
```

## 3. Tupla com único elemento

```python
tupla (5,) # virgula é obrigatória
```

Sem vírgula, o Python interpreta como inteiro:

```python
tupla = (5) # int, NÃO é tupla
```

---
# Acesso a Elementos da Tupla

O acesso é feito por **indexação**, assim como em listas e strings.

```python
cores = ("vermelho", "azul", "verde")
print(cores[1]) # azul
```

Também é possível usar indexação negativa.

```python
print(cores[-1]) # verde
```

---
# Imutabilidade das Tuplas

A principal diferença entre listas é a **imutabilidade**.

## Exemplo de erro

```python
cores[0] = "preto"
```

❌ Isso gera um erro:

```text
TypeError: 'tuple' object does not support item assignment
```

## O que PODE ser feito

```python
cores = ("vermelho", "azul")
cores = cores + ("verde")
```

Nesse caso, uma **nova tupla** é criada.

---
# Desempacotamento de tuplas

Desempacotamento permite atribuir os valores da tupla a várias variáveis.

## 1. Exemplo básico

```python
pessoa = ("Maria", 23)
nome, idade = pessoa
```

## 2. Usando o operador `*`

```python
valores = (1, 2, 3, 4, 5)
primeiro, *meio, uiltimo = valores
```

**Resultado:**

- `primeiro = 1`
- `meio = [2, 3, 4]`
- `ultimo = 5`

# Tuplas em Laços de Repetição

```python
pontos = [(1, 2), (3, 4), (5, 6)]

for x, y in pontos:
	print(f"x={x}, y={y}")
```

Muito comum em estrutura de dados mais complexas.

# Casos de uso de tuplas

## 1. Dados que não devem ser alterados

```python
coordenadas = (10.5, 20.8)
```

## 2. Retorno múltiplo de funções


```python
def calcular(a, b):
	return a + b, a * b
	
soma, produto = calcular(2, 3)
```

## 3. Chaves em dicionários

```python
localizacao = {(10, 20): "Entrada"}
```

Listas não podem ser usadas como chave.

## 4. Performance e segurança

Tuplas são:

- Mais rápidas que listas
- Protegidas contra alterações acidentais

Muito usadas em bibliotecas e código profissional.

# Exemplos Práticos

## Exemplo 1

Armano trabalha com a gestão de dois estoques de mercadorias que são representados como tuplas. Agora, ele precisa criar um relatório unificado com os produtos dos dois estoques juntos.

Para ajudá-lo, como você criaria um programa que **ler as informações dos estoques** e gera um relatório com **todos os produtos juntos**?

**Solução:**

```python
estoque1 = tuple(input("Produtos do estoque 1 (separados por vírgula): ").split(", "))

estoque2 = tuple(input("Produtos do estoque 2 (separados por vírgula): ").split(", "))

estoque_combinado = estoque1 + estoque2  
print(f"Estoque combinado:\n{estoque_combinado}")
```

# Quando usar Lista ou Tupla?

| **Situação**        | **Estrutura** |
| ------------------- | ------------- |
| Dados mutáveis      | Lista         |
| Dados fixos         | Tupla         |
| Chave de dicionário | Tupla         |
| Segurança           | Tupla         |

---
# Erros Comuns

❌ Esquecer a vírgula em tuplas de um elemento
❌ Tentar modificar valores da tupla
❌ Usar lista quando os dados não deveriam mudar

---
# Conclusão

Tuplas são ideais para representar **dados constantes**, melhorar a **segurança do código** e facilitar o **desempacotamento de valores**.

Dominar tuplas ajuda a escrever código mais limpo, eficiente e profissional.
